# hifi-block-doc-enrich

## 目标

把**短初稿**（例如 `docs/首页/section8-blocks/08-*.md` 仅有 stableKey、网格与一句话）扩成**可交给实现的说明**：能对照高保真（及用户提供的截图）还原视觉，并明确与模块化网格、Pinia、子模块的边界。

### 两种使用强度（按用户表述选用，未说明时默认「实现档」）

| 模式 | 适用场景 | 文档侧重 |
|------|----------|----------|
| **学习 / 一比一还原档** | 用户明确要「学习」「抄全」「保证做出来一模一样」「静态数据也要保留」 | **最大化信息密度**：稿面可见文案、表格/列表中的数字与单位、占位符、图例文字、空状态文案等**一律从 HTML 原文摘录**进文档；样式表**尽量穷尽**（含渐变 stops、`filter`、`mix-blend-mode`、`transform`、`z-index` 叠层）；资源与背景图**不省略**；伪代码节点附带**示例字面量**；必要时附「稿面 HTML 片段引用」便于核对。 |
| **实现档（默认）** | 日常开发交付 | 在可验收前提下保持可读：仍须满足下文**对齐契约**，但可对重复 token 做合并说明（须指向同一 CSS 规则）。 |

**核心约束**：文档里的**布局伪代码**、**样式/资源说明**、**学习档下的静态文案与数据**须与高保真在**可验收维度**上一致——按文档独立实现得到的页面，应与稿面在**结构顺序、可见节点、颜色字号间距圆角描边阴影、图片尺寸与路径、可见文字与数字**上对齐；若因网格裁剪或 absolute→flex 改写产生差异，必须在文档中**显式写明等价关系与取舍**，不得用「约」「类似」掩盖未核对项。

## 编辑前强制阅读（先于一切目标 `.md` 修改）

**在未完成本节清单前，禁止**对目标区块文档做任何扩写、删改或新建（允许仅 `Read` 初稿以确认路径与 stableKey）。智能体须用工具**实际打开**工程文件，不得凭记忆或片段猜测。

| 顺序 | 对象 | 「深入读取」最低标准 |
|------|------|----------------------|
| 1 | **参考 HTML** | 以标题文案 / `stableKey` / 已知 `class` 定位根节点后，**通读本模块负责展示的整段子树**（含嵌套、条件注释、内联 `style`、`<style>` 块若存在）；上下扩展至能确定**父级裁剪、兄弟装饰、脚本占位**为止，而非只读 grep 命中行上下各几十行就停。 |
| 2 | **关联 CSS** | 对子树中出现的**每一个** `class` / `id` / 可能影响本块的**祖先选择器**（含 `body`、`html`、通用布局 wrapper），在 `index.css`、`common.css` 及 HTML 引用的其它表（含 `@import` 指向的文件）中 **grep 并读完完整规则块**；含媒体查询、`@layer`、变量 `--*` 的定义与覆盖链；若规则依赖 `@font-face` / `url(...)`，须追到声明处。 |
| 3 | **静态文件** | 列出本子树内所有 `src` / `href`（字体、图、雪碧说明）及 CSS 中的 `url(...)` / `mask-image` / `content: url(...)`；对**每个**相对路径在导出目录内 **Glob / Read / 列目录** 核对文件存在；对栅格图、SVG 至少确认**文件名、扩展名、大致用途**（必要时 `Read` 图片旁 README 或同目录说明）；禁止在未核对存在性前把路径写入文档定稿表。 |
| 4 | **交叉收口** | 在脑中或草稿中建立「节点 → class → 规则文件与选择器 → 资源路径」对照后，再打开目标 `.md` 执行扩写。 |

若高保真目录结构异于常见 `index.html` + `index.css`，以**实际引用链**为准，把上述三类文件全部走通一遍后再写文档。

## 输入（用户需提供或可推断）

| 输入 | 说明 |
|------|------|
| 高保真工程路径 | 常见为 `ui/<导出目录>/index.html`，样式在 `index.css` / `common.css` |
| 目标区块 | 文案关键字、`stableKey`、或 `section*_` / `text_*` / `group_*` 类名 |
| 待补文档 | 多为 `docs/首页/section8-blocks/<basename>.md`，保留原有 YAML 式元信息行 |
| **设计截图（可选但强烈推荐）** | 用户提供的整页或局部 PNG/JPG/WebP；用于与 DOM/CSS **交叉验证**（层次、字重、渐变方向、装饰层是否漏抄） |

若用户只给目录，智能体自行打开 HTML/CSS 定位对应节点。若用户**仅提供截图**而无导出工程，须在文档中**明确标注**「以下样式数值无法从仓库 CSS 溯源，来自截图目测/待补工程」，并列出待用户补全的最小信息（如字体族、精确色值）；**禁止**在无工程文件时编造可度量数值并写成「已定稿」。

## 截图对照工作流（用户提供截图时必做）

1. **对齐范围**：确认截图对应 `index.html` 中哪一段子树（标题文案、角标、装饰是否与截图一致）；若截图与当前导出**不一致**，以用户说明为准（例如「以截图为准」），并在文档顶部写 **「截图与导出差异」** 表（项、截图表现、HTML 表现、文档采用哪一侧）。
2. **层次与遮挡**：对照截图核对 **z-index、半透明层、发光、内外阴影、模糊** 是否已在 §3 样式表与伪代码中体现；截图里「更亮/更暗」的条、带、角标，须在文档中对应到具体 class 或伪元素规则。
3. **文案与数据**：逐字核对截图中的标题、单位、数字、图例、小字说明是否与 HTML 文本节点一致；不一致则记入「截图 ↔ HTML」差异表，并写清实现应以何者为准。
4. **几何与间距**：用截图做**目视验收清单**（例如「主标题基线与副标题间距观感为单倍行高」）；凡能从 CSS 量化的，仍以 CSS 为准写入表格，截图用于发现漏项与导出错误。
5. **写入文档**：单设 **「截图核对结论」** 小节：列出已核对项、异常项、待产品确认项；学习档可附「截图标注建议」（用户在图上标 1/2/3 对应文档段落编号），便于评审。

## 高保真对齐契约（伪代码与说明必须遵守）

以下条款优先于文中其他「可选」表述；凡与契约冲突，以本节约束为准。

1. **DOM 顺序即伪代码顺序**：伪代码树中子节点自上而下顺序须与 `index.html` 中**本模块负责展示的子树**一致；不得调换图标与文字先后，除非稿面如此且已引用 HTML 片段证明。
2. **可度量数值须有 CSS 出处**：凡写入文档的 `px` / `rgba` / `hsla` / `%` / `deg` / `font-weight` / `letter-spacing` / `box-shadow` / `border` / `border-radius` / `opacity` / `backdrop-filter` / `filter` / `transform` / `gradient` 各 stop 等，须能在 `index.css`（或稿面带行号的内联样式）中**对应到具体选择器**；若样式来自父级继承，须写清「继承自 `.父级类` 的哪条规则」或注明稿面无独立规则则**禁止编造数值**。
3. **伪代码不是产品臆想**：伪代码块描述的是**设计稿已呈现或可推导的布局语义**，不是「未来可能加」的占位；若稿面无某控件，文档中不得作为本块必选子节点写入，除非单独标为 **「可选扩展（稿面无）」** 并说明与还原无关。
4. **从稿到 Vue 的改写须登记**：高保真常用 `position:absolute` + 大 `margin`/`left`/`top`。文档在写 flex/grid 替代方案时，必须补一句**视觉验收标准**（例如「标题字左缘与图标右缘间距仍为 4px」「条高度与父容器 100% 一致」），并列出仍须保持的**稿面数值**（padding、gap、图标 20×20 等）。
5. **资源一一对应**：每个 `img` / `background-image` / `mask-image` / `content: url(...)` 写明高保真相对路径、在伪代码中的节点名、**宽高校验**（HTML 属性或 CSS `width`/`height`/`background-size`/`object-fit`）；兄弟块的图不得并入本模块资源表。
6. **与仓库已有 SFC 对齐**：若 `src/components/` 已存在与 `stableKey` 或文档标题对应的 `.vue`，智能体须 `Read` 该文件，使文档 §6 骨架中的 **class 命名、props/emits、可访问性模式、CSS 变量名与默认值、选中态规则**与实现一致；若稿面与代码 intentionally 不同，在文档中写 **「与稿差异」** 小节列差异项，避免读者以为文档过时。
7. **视觉规范叠加**：大屏色板、发光、描边等若项目有统一规范，可引用 [dashview-ui-spec](../dashview-ui-spec/SKILL.md)；但当稿面数值与全局规范冲突时，**以稿面 CSS 为准**，并注明「全局 token 与稿不一致时采用稿值」。
8. **静态数据与可见文案（学习档强制，实现档按需）**：从本模块子树内摘录所有**用户可见**字符串（含 `title`/`aria-label`/图表轴标签/图例），以及表格、卡片、指标中的**数字、单位、百分比、日期**；以「原文 + 节点路径（class 或伪代码节点名）」表格呈现。若同一数据在多处重复，可合并行但须注明**所有出现位置**。禁止用「某某数据」概括而不给出稿面实际字面量。

## 工作流程

0. **完成「编辑前强制阅读」**：按上表完成对 HTML 子树、关联 CSS 全链、静态资源路径与文件的深入读取与核对；**未完成不得编辑目标 `.md`**。若初稿路径未知，可先只读初稿与高保真目录以确定目标文件，再回到本步补全阅读后再改文档。
1. **锁定 DOM**：在 `index.html` 中 `grep` 模块标题文案、`stableKey` 相关中文，或已知 `class`（如 `section_64`、`text_347`），在步骤 0 已通读子树的前提下，**标出本块精确起止行号**并保留引用用范围（必要时范围大于「上下各 30～80 行」以覆盖整块与邻接装饰边界）。
2. **抽取样式**：在步骤 0 已读完规则块的前提下，将须写入文档的度量整理成表；仍须 `grep` 复查是否有第二处覆盖规则或媒体查询内覆盖。记录并写入文档表格：`width`/`height`/`min-*`/`max-*`、`padding`/`margin`、`gap`、`color`、`font-size`/`line-height`/`letter-spacing`/`font-family`、`font-variant`、`text-transform`、`border`、`border-radius`、`background`（含渐变与多背景层序）、`box-shadow`（含 inset）、`opacity`、`display`/`flex-*`/`align-*`/`justify-*`、`grid-*`、`object-fit`、`object-position`、`white-space`、`word-break`、`text-shadow`、`overflow`、`clip-path`、`filter`、`backdrop-filter`、`transform`、`transition`。高保真常用绝对定位大 margin，文档中要翻译成 **flex / gap / 网格内语义**，并满足上文**改写须登记**。
3. **对照初稿**：读目标 `.md`，保留顶部「来源、stableKey、网格」等既有行，**在 `---` 之后**扩写，不擅自删用户已定约束。
4. **写清模块化映射**：说明该块在全幅稿中的角色，以及落到 `vue-grid-layout` 某一 `GridItem(x,y,w,h)` 后应保留/省略什么（例如顶栏多 Tab 拆成单列时只保留末项标题）；若裁剪导致与全稿像素级不同，写明**验收仍以单列容器内为准**。
5. **静态资源**：在步骤 0 已核对文件存在的前提下，写清高保真**相对路径**（如 `./img/FigmaDDSSlicePNG….png`），并一句说明落地到 `src/assets/...` 的惯例；装饰图与功能图标**分归属**，勿把兄弟节点的图写进本模块。细化资源目录时可再读项目技能 [dashview-asset-guide](../dashview-asset-guide/SKILL.md)。
6. **交互 default**：列出至少「纯展示 / 与全局 Tab 或 Store 联动」等伪代码分支；可点击时提醒无障碍（`button` / `role="tab"`）。分支行为须与**稿面是否可点**、**产品说明**一致；稿面静态字而文档写点击，须标为产品扩展并给出 `emit`/Store 字段名（与现有 store 字段对齐时 `Grep` `dashboardStore.js`）。
7. **Vue 单文件骨架（与高保真对齐版）**：§6 给出的伪代码骨架须满足：**模板节点层级与 §2 伪代码树一致**；`scoped` 样式中的可度量值与 §3 表格一致（可用 CSS 变量但默认值须等于稿面抽取值）；**禁止**在骨架里写与 §3 矛盾的占位数值。若项目已有同功能组件，骨架应与其**同构**（仅省略非必要 import 路径细节时可注明「与 `Foo.vue` 一致」）。
8. **若有截图**：执行上文 **「截图对照工作流」**，并把结论写入文档固定小节。

智能体须**自行执行** `Read` / `Grep` / `Glob`（及必要时终端列目录）完成对照，勿只给操作建议。**默认顺序**：先完成对 HTML + CSS + 静态文件的深入阅读（见「编辑前强制阅读」），再修改目标 `.md`。

## 布局伪代码格式规范（须贴近实现、且可对照稿）

推荐使用带**缩进树**与**行内样式注解**的 `pseudo` 代码块，且每一叶子节点至少标注与稿面相关的度量（从 CSS 抄录，不猜）。**学习档**下鼓励在节点后附带**稿面示例字面量**（从 HTML 抄录，短句可全文，长文可首尾各 20 字 + 中间省略号并注明总长度）。

````markdown
```pseudo
component ExampleBlock:
  root: <header class="example-block">   // display:flex; align-items:center; width:100%; height:100%; padding:0 12px; box-sizing:border-box;
    mark: <img class="example-block__mark" />   // 20×20; margin-right:4px; object-fit:contain; src → §4
    title: <span class="example-block__text">会议调度</span>   // color/font-size/line-height/letter-spacing/white-space → §3 表；字面量「会议调度」→ §3.5
  state:
    .is-active → box-shadow / text-shadow → §3 表（须附 index.css 选择器）
```
````

**禁止**：伪代码里只有「横向一条」而无具体 flex 属性；禁止省略 §3 已存在的 `letter-spacing`、`text-shadow`、`inset` 阴影、`filter`、`渐变 stop` 等「细但影响还原」的项。

## 输出文档推荐结构（贴入目标 `.md`）

以下块顺序可微调，标题可改为用户仓库惯用中文。

1. **设计意图**：高保真中的位置、与邻块关系；模块化后的语义是否收窄。
2. **布局（伪代码）**：符合上文**格式规范**的 `component` / `root` / 子节点树、对齐方式；单列时如何用 flex 替代稿面 absolute，并附**视觉验收**短列表。
3. **颜色与排版**：表格列出字色、字号、行高、字间距、圆角、描边、字体族与 Web 降级；**每行附 CSS 选择器或「继承说明」**。
4. **静态资源**：每个 `img` 的导出文件名、在高保真目录下的路径、是否建议重命名后入 `src/assets`；**尺寸来源**（属性或 CSS）。
5. **可见文案与静态数据（学习档必选，实现档有则必写）**：表格列「原文」「节点/class」「是否多处出现」「建议 Vue 落地（prop/常量/mock）」；列表、表格、图表刻度如有**固定示例数字**，逐行摘录。
6. **交互与状态**：枚举、`emit`、Store 字段、选中态 class；无交互则明确写「仅分区标题」。
7. **Vue 单文件骨架（伪代码）**：须与 §2、§3 同源；若仓库已有对应 `.vue`，写明「与 `src/components/.../Xxx.vue` 对齐」并列出 intentionally 差异（若有）。
8. **与 Store / 邻块关系**：指向对应 `mod-*`、`title-*` 或 `模块化初始页面描述.md` 链接。
9. **（推荐）高保真 ↔ 实现核对表**：三列「稿面（class/规则）」「文档取值」「实现文件行号或待办」逐项打勾，减少文档与代码漂移。
10. **（有截图时必选）截图核对结论**：已对齐项、差异项、采用口径（以截图 / 以 HTML / 待确认）。
