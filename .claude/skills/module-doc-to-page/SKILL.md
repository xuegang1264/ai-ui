# module-doc-to-page · 按模块文档开发/重构页面

## 与相邻技能的分工

| 技能 | 何时用 |
|------|--------|
| **本技能** | 文档已能指导实现（含 § 布局/样式/文案/资源/交互骨架）；目标是把 **SFC + 注册 + 资源** 做到与文档一致。 |
| `.agents/skills/hifi-block-doc-enrich` | 文档过短或缺高保真对齐；需先**扩写/校正 `.md`** 再实现。 |
| `.agents/skills/init-doc-section8-split` | 需把「§8 逐区」拆成多块 `.md` 与索引，而非直接写组件。 |

若用户只丢 stableKey 或一句需求，而目标 `.md` 仍像占位：先判断是否应触发 **hifi-block-doc-enrich** 补文档，再进入本流程。

## 强制顺序（写代码前）

1. **读透目标文档**：`stableKey`、网格 `(x,y,w,h)`、`props.title` 等与壳层关系、禁止纳入本块的节点（如「不含 section_65」）、与邻块/标题条分工（如 `title-*` vs `mod-*`）。
2. **对照高保真（若文档引用）**：文档给出 `index.html` 锚段 / `index.css` 选择器时，用工具 **Read/grep** 打开对应子树与规则，**禁止**仅凭文档二次摘要猜数值。
3. **定位现有实现**：`grep stableKey`、`Read src/components/stepOne/widgetMap.js`；存在则 **重构** 该 SFC（及仅因契约变化而需改的父级/Dashboard 壳层），避免平行 duplicate 组件。
4. **资源**：文档 § 静态资源须落地到 `src/assets/...`（或项目既有约定目录）；**禁止**在生产路径长期 `import`/`url()` 依赖 `ui/lanhu_*`。拷贝后在本组件或并列 `*-assets.js` 集中引用。
5. **壳层**：`DashboardModule` 的 `module-body` 内边距、`props.title` 与模块自带顶栏是否重复，须与文档「列头由 title-* 承担」等说明一致；必要时用 `margin`/`height` 补偿或调整 `node.props`，并在 PR/提交说明里写一句。

## 实现契约（与文档可验收一致）

- **DOM 顺序**：与文档 § 伪代码或 HTML 锚段顺序一致；为单列/网格改写时，在代码注释或提交说明中写清**等价关系**（如 absolute → Grid）。
- **样式**：文档表格或摘录中的 **色值、字号、行高、圆角、描边、渐变、尺寸** 优先逐条落地；全局 token 与稿冲突时**以模块文档/稿为准**并注明。
- **文案与静态数据**：与文档 §「可见文案与静态数据」一致；占位符、计数、`0/17` 等字面量不得随意替换（除非文档标明可配置）。
- **交互**：文档 § 交互与 emits 列出的行为，用 `<button>` / `role`+键盘 / 真实 `checkbox` 等满足无障碍最低要求；**未在文档中承诺的**功能不擅自扩展。
- **Store**：文档写明与 `meetingStore` 等形状不一致时，要么扩展 store 并写明字段，要么新建 store并在实现旁注释边界；**不把无关 store 硬塞进组件**。

## 新建 vs 重构

| 情况 | 动作 |
|------|------|
| `widgetMap` 无该 `stableKey` | 新建 SFC → 注册 `widgetMap` → 若需大屏格子则在 `dashboardStore`（或唯一布局源）增加节点，`props`/`layout` 与文档一致。 |
| 已有占位或旧实现 | **替换/重构**为文档目标结构；删除死代码与错误 class；更新资源引用。 |

## 完成后自检（提交前）

自己检查下代码有无问题，无需运行yarn进行判断

## 反模式

- 文档要求「本块不含顶栏标题」却在组件根再写一遍标题字。
- 未读文档 §「禁止误用」区块，把 KPI 条、全页顶栏等无关节点做进模块。
- 大段从 Lanhu **粘贴**全局 class 到非 scoped 样式，污染全站。
- 文档未要求的「顺手重构」跨目录扩散。
