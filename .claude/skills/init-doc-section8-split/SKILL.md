# init-doc-section8-split

## 目标

把源文档中 **§8 逐区文字说明**（从 `## 8. 逐区文字说明` 起到 `## 9.` 之前）拆成**便于逐块实现**的多个文件；每个文件尽量对应**一条完整读屏/实现动线**（列标题 + 其下内容，或标题条 + 正下方内容区），避免把强关联的标题说明与正文拆成两个孤立文件。

## 配对原则（标题区 + 内容区）

执行拆分时**优先按语义与布局成组**，再决定 basename：

1. **1:1 标题条 + 内容格**（§3、§7 中 `title-*` 网格紧邻其下唯一 `mod-*`）  
   - 将 **`title-*` 段落**与**紧随其后的 `mod-*` 段落**写入**同一文件**，中间用 `---` 分隔；文件名沿用**内容区**的 `08-mod-*` basename（见下表），便于与组件/Store 主键对齐。
2. **列级标题条 + 列内多块内容**（§4 `title-njtg`、§6 `title-nsfw`）  
   - 列标题只概括整列：把该 **`title-*` 段**与 §8 中**该列下列出的全部内容块**（各 `mod-*` 及其卡片内标题/内容）放进**同一文件**，自上而下保持 §8 原文顺序；basename 使用 `08-col-*`（见下表）。
3. **卡片内已含「标题 + 内容区」两段**（§8.3、§8.4、§8.5 中同一 `stableKey` 下的「标题」「内容区」）  
   - 本来就在连续段落中，**保持在一个 `08-mod-*.md` 内**，勿再拆成两个文件。
4. **无配对列标题的块**（如 `sitg`、`mod-map`，或中间列预警/结论）  
   - 仍**单文件单块**；若某段在源文中分为多小节，合并进同一文件，中间用 `---` 分隔。

若用户明确要求「只要标题」或「只要某 mod、不要列头」等粒度，可按其 basename 列表单独生成，但须在回复中说明与默认配对策略的差异。

## 源与输出

| 项 | 路径 |
|----|------|
| 源文档 | `docs/首页/模块化初始页面描述.md` |
| 输出目录 | `docs/首页/section8-blocks/`（若不存在则创建） |
| 索引 | `docs/首页/section8-blocks/_index.md` |

## 区块与文件命名（固定清单）

按下列顺序生成文件；**文件名**使用表格中的 `basename`（不含 `.md` 时加 `.md`）。  
与「标题 + 内容」合并相关的项在「包含 / stableKey」列中写全。

| basename | 包含（源 §8 中截取范围） | stableKey / 备注 |
|----------|--------------------------|------------------|
| `08-page-header` | 页眉主标题段落 | 非 Store，见 `DashboardView.vue` |
| `08-mod-uav` | **`title-uav` 标题条说明** + **`mod-uav` 内容区** | `title-uav`，`mod-uav` |
| `08-mod-njzx` | **`title-njzx` 标题条说明** + **`mod-njzx` 内容区** | `title-njzx`，`mod-njzx` |
| `08-col-njtg` | **`title-njtg` 列标题** + §8.3 五项（各卡「标题」+「内容区」全文） | `title-njtg`，`mod-njzxjsgk`，`mod-schedule`，`mod-build`，`mod-service`，`mod-eval` |
| `08-sitg` | `sitg` 顶态势条 | `sitg` |
| `08-mod-map` | 地图主视窗 | `mod-map` |
| `08-mod-warning` | 「预警信息」标题 + 内容区（同格） | `mod-warning` |
| `08-mod-conclusion` | 「综合分析结论」标题 + 内容区（同格） | `mod-conclusion` |
| `08-col-nsfw` | **`title-nsfw` 列标题** + `mod-nszx` + `mod-ns-ability`（各含卡片标题与内容区） | `title-nsfw`，`mod-nszx`，`mod-ns-ability` |
| `08-mod-nsservice` | **`title-nsservice` 标题条** + **`mod-nsservice` 大块内容区** | `title-nsservice`，`mod-nsservice` |

共 **10** 个区块文件（§8 语义全覆盖；§8 内 `---` 仅作排版，拆分时按上表归入对应文件，组内仍可用 `---` 分隔大段）。

> **与旧版拆分的差异**：不再单独生成 `08-title-uav.md`、`08-title-njzx.md` 等与对应 `08-mod-*` 脱节的纯标题文件；第二列、第四列用 `08-col-njtg`、`08-col-nsfw` 承载列标题与列内所有模块说明。若仓库中仍有旧文件，合并后应删除冗余 `08-title-*.md`（页眉除外无 `08-title-page`）并更新 `_index.md`。

## 单文件模板

每个 `docs/首页/section8-blocks/{basename}.md` 使用：

```markdown
# {中文短标题：建议体现「标题+内容」或列名，与源文一致或略作归纳}

- **来源**: `docs/首页/模块化初始页面描述.md` → §8 逐区文字说明
- **stableKey**: `{单个 key，或逗号分隔多个，或 bulleted 多行}`
- **网格 (x, y, w, h)**: `{主实现块的坐标；若多格可写「见正文各段」或分列 bullet}`

---

{从源文档 §8 拷贝的本文件应承担的全部正文，保持原有加粗、列表与段落，不删改语义；组内大段之间可用 ---}
```

- **多 stableKey 文件**（如 `08-col-njtg`）：YAML 式列表或一行内用顿号/逗号列出均可，索引 `_index.md` 中应逐项标出坐标（可从子段落抄）。
- 不在单文件内重复 §1～§7 表格；需要时只写「详见源文档 §x」一行链接：`[模块化初始页面描述](../模块化初始页面描述.md)`。

## `_index.md` 内容

生成或更新 `docs/首页/section8-blocks/_index.md`：

1. 一级标题：`§8 区块索引（按开发顺序）`
2. 有序列表：每一项为 `[basename]({basename}.md)` + 一行括号内列出本文件涉及的全部 `stableKey` 与各自 `(x,y,w,h)`（可从子段落或 §3～§7 表抄）。
3. 末尾注明：源文档路径与「拆分日期」可用当天日期（用户消息中的 Today 若可得）。
4. 列表项数与上表 **10** 个 basename 一致；若用户仅增量生成部分文件，已生成项写实链接，未生成项可标注「待生成」或暂不列入。

## 执行步骤（智能体）

1. 读取 `docs/首页/模块化初始页面描述.md`，定位 `## 8. 逐区文字说明` 与 `## 9. 模块数量口径说明`。
2. 按上表 **basename** 从 §8 中截取对应段落；边界以 `**...stableKey...**`、`**标题「...」**`、`### 8.x` 小节标题为参照，**优先按「配对原则」把标题条与内容区拷入同一文件**，不确定时对照 §3～§7 表格中的 `stableKey` 与坐标。
3. 写入 `section8-blocks/*.md` 与 `_index.md`；已存在且内容相同的文件可跳过，用户明确要求「覆盖」时再全文替换。若正在从旧版（20 文件、标题与 mod 分离）迁移，删除已被合并吸收的孤立 `08-title-*.md`（`08-page-header` 保留）并更新索引。
4. 回复用户：输出目录、文件个数、配对策略摘要、如何从索引打开下一块。

## 与用户协作

- 用户若只要其中几块：只生成其点名的 `basename` 列表，仍更新 `_index` 中对应项或备注。
- 源文档若今后移动：先更新本 SKILL 中路径，再执行拆分。
